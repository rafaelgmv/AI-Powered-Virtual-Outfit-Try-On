1. AI now returns JSON with two separate parts: a ~200-word user summary (Style Identity + Color Palette + Outfit Formulas) and hidden per-category search queries (tops/bottoms/shoes/outerwear) fed verbatim to the scraper. Removed the old lossy keyword pipeline (which only used 3 generic keywords from one section), so results now match the inspiration image (e.g. LeBron photo -> NBA/Lakers items) and the UI no longer shows a wall of text.

2. Rendered the full style description as actual markdown instead of raw text. Added a small dependency-free renderer (## headings, * bullets, **bold**, paragraphs) so the summary shows styled headings and bullet lists; edit mode still shows raw markdown source.

3. Prioritized trustworthy retailers by sorting all scraper results once by a review-weighted (Bayesian) rating, so a 5-star/2-review reseller can't outrank a 4.4-star/12k-review retailer. Thin-review products drift toward the batch average instead of the top; unrated products are placed last by design. No retailer allowlist and no extra API calls; display of stars is unchanged.

4. Removed the Edit button (and Save/Cancel) from the full style description. Editing it did nothing useful since the scraper runs off the hidden queries, not the summary text, so the profile is now read-only. Cleaned up the dead state, handlers, and unused icon imports.

5. Hero/title polish: removed the old "Personal Styling AI" pill tag and the dead "Get Started" button; added a centered gradient app title at the top of the page ("Personalized Styling AI") with the descender-clipping fixed; tightened the gap between the headline lines and reworded the subtitle to "See it on you before buying." Also set the browser tab title/metadata to "Personalized Styling AI" (there was none before) and fixed the page lang from "pt" to "en". Cleaned up the unused icon imports.

6. Added a Portrait upload field on step 1 (below Clothing Sizes). The uploaded portrait now drives the Avatar Preview panel on the right, replacing the old "3D Avatar Placeholder" with the user's image (falling back to a "Waiting for user portrait..." placeholder when none). Also renamed step 3's "Inspiration" heading to "Reference Photos".

7. Added a "Choose preset" option to the step-1 Portrait field (tabbed with Upload) so users who don't want to upload a photo can pick a real model portrait instead. Presets cover a diversity spread (ethnicity x gender) and each shows its category label + name. The whole preset list is data-driven from /public/presets/presets.csv (columns: file,ethnicity,gender,name) parsed at runtime — adding a preset is just dropping an image and adding a CSV row, no code changes. Portrait is now required to leave step 1 (upload or preset), with a helper hint while none is chosen.

8. Made presets auto-fill and lock the build. Each preset row in presets.csv now also carries height/weight/bodyType; selecting a preset fills those step-1 fields with the model's build and disables them (with a hint), so a "fat build + skinny model" mismatch can't happen. Deselecting the preset (or switching to Upload) restores the user's own typed values. Clothing sizes are never locked — they stay the user's since they drive the product search.

9. Avatar Preview / preset-grid polish. The chosen portrait now sits inside the rounded placeholder card with object-cover (so every image, regardless of proportions, frames identically) and the label below shows the chosen model's name (or "Your portrait" for uploads). Capped the preset grid to a scrollable box and gave scroll regions a subtle dark themed scrollbar instead of the default white one.

10. Made Reference Photos optional and feed the portrait to the AI directly. Since the portrait is mandatory, it's always sent to the model as a visual cue, so a new user can skip reference photos entirely and still get a profile from their portrait + selected style + notes. When references are added they're sent too. Added a soft "notes-first" priority to the prompt: the selected style and written notes are authoritative and win when an image conflicts; images lead only when the user gives little written direction. Allowed /presets/ images (not just /uploads/) to be read by the AI so preset portraits work as input.

11. Handled the portrait-vs-reference conflict case (e.g. Zendaya portrait + LeBron reference). The portrait and reference photos are now sent as separate, explicitly labeled inputs (portrait first = the person; rest = style inspiration) so the AI stops guessing which is which. By default it blends them as equal style cues, but if the references are severely inconsistent with the portrait it disregards the references entirely, styles from the portrait alone, and says so in the Style Identity section of the profile.

12. Wired up the step 3 -> shop -> outfit selection flow. Step 3's previously-dead Continue button is now "View Recommendations" and smooth-scrolls to the shop section. Each product card has an explicit "Select" / "Selected" button (clicking the card no longer hijacks the link), letting the user pick one item per category. A sticky outfit bar shows Tops/Bottoms/Shoes/Outerwear slots filling in, and a "Generate Preview" button enables only once all four are chosen. The button currently gathers the chosen outfit and stubs the call (image-generation model not built yet).

13. Avatar Preview polish to reserve space for the upcoming try-on output. Once a style profile is generated, two stacked placeholder slots ("Front view" / "Side view", same size/shape as the portrait) appear below the portrait, ready to hold the generated side profiles. Made the preview panel stretch to match the form column height so there's no dead gap, and the label under the portrait switches from the model's name to "Portrait" once the profile is generated.

14. Chose the try-on image generation backend. First attempted Gemini image generation (gemini-2.5-flash-image) but the free tier has a hard limit of 0 requests for the image model (confirmed via a direct 429: "limit: 0" on every quota metric) — it requires a billed Google Cloud project. Pivoted to CatVTON, a virtual try-on diffusion model that runs locally on an NVIDIA GPU (chosen over IDM-VTON for lower VRAM use). Got CatVTON's demo running locally, fixing two dependency issues in its install along the way (unpinned diffusers from GitHub main being too new for the pinned torch 2.4.0 -> pin diffusers==0.31.0; that then needing a newer accelerate -> pin accelerate==0.34.2). Full local install instructions, including both fixes, are in CATVTON_SETUP.md. Next step (not yet built): wrap CatVTON in a small Python service and rewire the "Generate Preview" flow to call it (one front try-on image from top+bottom+outerwear; shoes shown but not selectable; no side view since the model can't produce one).

15. Integrated CatVTON for the front try-on. Added a FastAPI service (vton_service.py in the CatVTON folder) that loads CatVTON once and exposes POST /tryon (person + one garment + cloth_type -> result image), mirroring the demo's pipeline call. Rewired the Next.js side (lib/avatarGeneration.ts) to drop Gemini and call that local service over HTTP in two sequential passes: upper (outerwear if selected, else top) then lower (bottom), producing one front image. The Front view slot now shows the real try-on; the Side view slot stays a placeholder (CatVTON can't generate a new pose). Fixed the selection gate to match reality: only Tops + Bottoms are required to generate (Outerwear optional, Shoes excluded from the preview entirely). Running it needs two processes (the Python service + npm run dev) — see CATVTON_SETUP.md.

16. Added a personalization/cache system so returning users skip the cold start. The full profile (body data, sizes, portrait, selected style, style notes, reference image URLs, and the generated description + hidden queries) is saved to the browser's localStorage (lib/profileCache.ts — no backend/accounts). On a return visit the form restores everything and lands the user on step 2 with their style notes pre-filled and editable; step 1 stays reachable via Back. Reworked step 3's reference photos into a thumbnail gallery: each image has an X to remove, an "Add more" tile appends (capped at 2 photos, tile hides at the cap), and the set is cached. The generate button reads "Regenerate" once a profile exists. Added a red-gradient "Start over" control (with a Confirm/Cancel step) that wipes the cache, resets the form to step 1, and clears all generated output — style profile, shop results, and the front-view try-on image.
